168 ## 3.0.0 beta 1 (2011-08-14)
169 
170 Breaking Changes:
171 * Removed overloads of logging methods that were taking format string from ILogger and ILogger and IExtendedLogger and didn't have word Format in their name.
172  * For example:
173  * void Error(string format, params object[] args); // was removed
174  * void ErrorFormat(string format, params object[] args); //use this one instead
175  * impact - low
176  * fixability - medium
177  * description - To minimize confusion and duplication those methods were removed.
178  * fix - Use methods that have explicit "Format" word in their name and same signature.
179 * Removed WebLogger and WebLoggerFactory
180  * impact - low
181  * fixability - medium
182  * description - To minimize management overhead the classes were removed so that only single Client Profile version of Castle.Core can be distributed.
183  * fix - You can use NLog or Log4Net web logger integration, or reuse implementation of existing web logger and use it as a custom logger.
184 * Removed obsolete overload of ProxyGenerator.CreateClassProxy
185  * impact - low
186  * fixability - trivial
187  * description - Deprecated overload of ProxyGenerator.CreateClassProxy was removed to keep the method consistent with other methods and to remove confusion
188  * fix - whenever removed overload was used, use one of the other overloads.
189 * IProxyGenerationHook.NonVirtualMemberNotification method was renamed
190  * impact - high
191  * fixability - easy
192  * description - to accommodate class proxies with target method NonVirtualMemberNotification on IProxyGenerationHook type was renamed to more accurate
193  NonProxyableMemberNotification since for class proxies with target not just methods but also fields and other member that break the abstraction will
194  be passed to this method.
195  * fix - whenever NonVirtualMemberNotification is used/implemented change the method name to
196  NonProxyableMemberNotification. Implementors should also accommodate possibility that not
197  only MethodInfos will be passed as method's second parameter.
198 * DynamicProxy will now allow to intercept members of System.Object
199  * impact - very low
200  * fixability - easy
201  * description - to allow scenarios like mocking of System.Object members, DynamicProxy will not
202  disallow proxying of these methods anymore. AllMethodsHook (default IProxyGenerationHook)
203  will still filter them out though.
204  * fix - whenever custom IProxyGenerationHook is used, user should account for System.Object's
205  members being now passed to ShouldInterceptMethod and NonVirtualMemberNotification methods
206  and if neccessary update the code to handle them appropriately.
